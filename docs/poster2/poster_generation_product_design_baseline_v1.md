# 海报可控生成产品设计基线 v1

## 1. 文档定位

本文档作为 poster2 方向的基础产品设计基线，用于统一产品、架构、工程与运营对“海报可控生成”的正式理解。

本文档不讨论具体代码实现，也不承载阶段性调试细节，而是用于锁定以下正式口径：

- 产品本质
- 两类模板家族
- 三层逻辑
- 稳定性控制边界
- 当前阶段的工程优先级

---

## 2. 产品本质

我们的完整业务链路是：

> 输入素材 → 生成海报文案 → 生成海报 → 发送邮件

其中，真正决定产品壁垒与交付质量的核心环节，不是文案生成，也不是邮件发送，而是：

> **海报的稳定可控生成能力**

海报生成能力的目标不是“偶尔出一张好看的图”，而是长期满足以下四个基本约束：

- 产品不变形
- 文案确定性
- 结构稳定
- 审美基本在线

因此，poster2 的核心不是自由拼图，不是一次性样本优化，而是：

> **建立一套可控、可复现、可审查、可运营的海报生成体系**

---

## 3. 两类模板家族的正式定义

模板体系不定义为大量离散模板，而定义为：

> **一套共同模板语法下的两个模板家族**

它们属于同一生成体系，而不是两套独立系统。

### 3.1 模板家族 A：营销讲解型海报

英文名：`Campaign Explainer Poster`

核心目标：

> 先吸引，再解释，再带出系列感。

典型特征：

- 强品牌头部
- 大标题 / 强主张
- 主产品 + 场景共同组成 hero 区
- feature callout / 引导线卖点
- 底部系列产品条或补充展示区

适用场景：

- 邮件首屏主视觉
- 电商 / 活动推广图
- 新品推广页
- 品牌讲解型营销图

### 3.2 模板家族 B：产品图录型海报

英文名：`Product Sheet / Product Story Poster`

核心目标：

> 先认产品，再读信息，再促转化。

典型特征：

- 顶部品牌横幅或品牌锚点
- 产品参考号 / 型号 / 系列归属
- 单品主图或主推组合图
- 参数信息块 / 规格块 / 说明文案
- CTA / 联系我们 / 联系方式
- 品牌 footer 或联系收束区

适用场景：

- 产品图册
- 经销商资料
- 邮件详情页
- 产品说明页
- 纵向长图 / 移动端产品页

---

## 4. 两类模板的共同原则

虽然存在两个模板家族，但它们必须共享同一套底层原则。

### 4.1 共同输入

两类模板都基于以下输入资产运行：

- 品牌资产
- 产品主图
- 可选场景图
- 功能点 / 卖点文案
- 系列图 / packshot
- 参数信息 / reference code
- CTA / 联系信息

### 4.2 共同输出

两类模板都必须输出：

- 最终海报图
- 模板版本信息
- 引擎路径信息
- degraded / fallback 状态
- 结构完整性解释
- 调试与审查元数据

### 4.3 共同底线

无论属于哪一类模板，以下约束都不能被突破：

- 产品主图不得变形
- 品牌资产不得漂移
- 文案内容不得自由改变结构含义
- 输出结果必须可复现到同等级
- fallback 必须可见、可解释、可接受或可判失败

---

## 5. 三层逻辑的正式定义

海报生成逻辑正式定义为三层：

> **结构层 → 控制层 → 美化层**

这三层是模板治理逻辑，而不是视觉效果叠加顺序。

### 5.1 第一层：结构层

英文名：`Structure Layer`

职责：

> 定义这张海报由哪些区域组成，以及每个区域负责什么。

它解决的是：

- 区域划分
- 图层顺序
- 文本槽位
- 标题 / 副标题 / CTA 的归属
- 产品区、场景区、参数区、底部区的存在方式
- 哪些是 mandatory region，哪些是 optional region

原则：

> **结构层必须由程序确定性控制，不允许模型自由决定结构。**

### 5.2 第二层：控制层

英文名：`Control Layer`

职责：

> 让结构在真实输入波动下仍然稳定工作。

它解决的是：

- 区域是否完整成立
- 哪些 slot 可以 collapse
- 输入尺寸、比例、留白、文案长度不理想时如何处理
- fallback 是否可接受
- degraded 是否应判为失败
- 文本如何截断、缩放、换行
- 产品图如何 fit / contain / crop
- 系列图不足时如何折叠或补齐
- 引擎路径如何选择与回退

控制层应分两阶段理解：

#### 第一阶段：稳定性控制

先解决：

- 结构是否成立
- 输入是否可治理
- fallback 是否可接受
- 结果是否可解释

#### 第二阶段：布局控制

后续再解决：

- region 内部布局行为
- 文本预算
- fitting policy
- peer-region 平衡
- 更细的排版模式

原则：

> **控制层先追求稳定、可预测、可复盘，再追求更聪明的布局。**

### 5.3 第三层：美化层

英文名：`Beautification Layer`

职责：

> 在结构与控制已经稳定后，统一视觉语言，让结果审美基本在线。

它解决的是：

- 字体语言
- 圆角体系
- 阴影体系
- 背景材质感
- 色彩层级
- 标题视觉张力
- CTA 的品牌感
- 画面节奏与高级感

原则：

> **美化层必须后置，不能用来掩盖结构或控制层问题。**

---

## 6. 稳定性控制边界的正式定义

当前阶段不直接进入“布局细调”，而先建立：

> **稳定性控制边界**

它属于控制层第一阶段，是当前工程的主任务。

### 6.1 稳定性控制必须管理的内容

#### A. 结构完整性

必须明确：

- 哪些 region 是必显
- 哪些 region 可折叠
- 折叠条件是什么
- 折叠后是否补位
- 是否允许部分结构成立

#### B. 输入完整性

必须对输入先做可治理判断，而不是直接盲排：

- 文本长度等级
- 图片比例等级
- 图片留白等级
- gallery 完整度等级
- 参数信息完整度等级

#### C. 渲染完整性

必须定义：

- 何时算 Puppeteer 成功
- 何时允许 Pillow fallback
- 哪些模板 fallback 后仍可交付
- 哪些模板 fallback 后应判为结构不完整

#### D. 结果解释性

必须输出：

- 使用了哪个模板
- 走了哪个 renderer
- 是否 degraded
- 哪些区域成功注入
- 哪些区域 collapse
- 最终结果为何如此

### 6.2 稳定性控制当前不管理的内容

当前阶段的稳定性控制不负责：

- 追求更高级的视觉氛围
- 微调阴影与透明层
- 样本级 CSS 补丁
- 用遮罩掩盖结构问题
- 为单一样本做特例化视觉修补

这些属于后续布局优化或美化层任务。

---

## 7. 两类模板在三层逻辑下的落位

### 7.1 模板家族 A：营销讲解型

#### 结构层

必须优先锁定：

- 品牌头部
- hero 产品区
- 场景区
- feature / callout 区
- 标题带
- 底部系列图区

#### 控制层

必须优先控制：

- header lane 稳定
- feature 数量模式
- scenario / product 的 peer relationship
- gallery strip 的数量与折叠规则
- title / subtitle 的占位预算

#### 美化层

最后才处理：

- 主视觉冲击力
- 引导线与 callout 的品牌风格
- 标题带与底部条的节奏
- 背景与前景的层次感

### 7.2 模板家族 B：产品图录型

#### 结构层

必须优先锁定：

- 品牌横幅 / 品牌锚点
- reference / 型号区
- 单品主图区
- 参数区 / 说明区
- CTA 区
- footer 品牌区

#### 控制层

必须优先控制：

- 产品主图不变形
- 参数区与正文区不漂移
- CTA 与品牌 footer 不互相抢位
- reference / 型号 / 价格 / 联系信息的确定性位置
- 长图纵向节奏稳定

#### 美化层

最后才处理：

- 图录感
- 专业感
- 信息秩序
- CTA 的信任感
- footer 的品牌质感

---

## 8. 当前阶段的正式工程口径

当前阶段正式口径如下：

> 我们的海报生成系统，不是自由拼图，也不是单次样本优化，而是“一套共同模板语法下的两个模板家族”。模板治理按“结构层、控制层、美化层”三层推进。当前阶段只推进结构层固化与控制层第一阶段——稳定性控制，不进入大规模布局细调与视觉美化。核心目标是先达成：产品不变形、文案确定性、结构稳定、fallback 可解释、结果可审查。

这意味着当前工程优先级是：

1. 先锁模板结构
2. 再锁稳定性控制边界
3. 再进入布局控制
4. 最后再进入美化层

---

## 9. 当前基础产品设计基线

从产品层面，海报生成能力当前的基础设计基线定义为：

### 9.1 核心目标

- 稳定可控生成
- 产品不变形
- 文案确定性
- 结构稳定
- 审美基本在线

### 9.2 模板策略

- 一套共同模板语法
- 两个模板家族
- 统一元数据与结果解释
- 统一 fallback / degraded 语义

### 9.3 工程策略

- Structure-first
- Stability-before-layout
- Layout-before-beauty
- Explainable outputs
- Dual-renderer compatibility

---

## 10. 一句话正式定义

> **海报生成的核心不是“生成一张图”，而是在两类模板家族下，用结构层保证版面确定性，用控制层保证输入波动下的稳定性，用美化层保证结果基本在线，最终形成一套可控、可复现、可运营、可扩展的海报生成体系。**
